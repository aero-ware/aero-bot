Setters

This collection of commands has commands that set important information into the database and ALL require ADMINISTRATOR permissions

setlogchannel

This command sets the logging channel for the server where important events are logged to. It is recommended that it be a private channel reserved for staff.
You can optionally say 'none' as the argument to unset the log channel. Aliases: setlogchannel, setmodlog, logchannel
Expected Arguments: <channel>
Example Usage: >setlogchannel #logs
That should be a channel mention and not just the channel name.

banpurgedays

This command set the days of messages will be delete when a member is banned. The maximum days of messages you can delete is 7. Aliases: banmessagedeletedays, banpurgedays, messagedeleteonban
Excpected Arguments: <days( maximum 7)>
Example Usage: >banpurgedays 5

setblacklist

This command can add or remove words considered part of this server's swear words list. By default, it has no words. Use 'add' or 'remove' as the first argument and the next argument will be a full string.
For example. If you used >setblacklist add a b c x y z, the string 'a b c x y z' will be added to the database and NOT each word separately. This is to increase flexibility for the end user. Aliases: swearwords, blacklistedwords, wordblacklist, setblacklist
Expected Arguments: <add/remove> <string>
Example Usage: >setblacklist add apple

setadchannels

This command allows you to add or remove channels where non admins are allowed to send invites to servers that is not the current server
Aliases: setadchannels, adchannels, allowads
Expected Arguments: <add/remove> <channel>
Example Usage: >setadchannels add #advertising
Note: please mention the channel

setwelcomemessage

This command sets the welcome messsage for the server, it will be sent when anyone joins the server.
Aliases: welcome, joinmessage, welcomemessage, setwelcome, setwelcomemessage
Expected Arguments: [channel] <message>
Example Usage >setwelcome #joins Welcome <@> to this server!
Note, use <@> to specify a mention of the joining member, it can be used multiple times in the same message. If channel is not mentioned as the first argument, it is assumed that the target channel is the current channel.

setmutedrole

This command sets the muted role for the server. Used for the mute and unmute commands.
Aliases: setmutedrole, setmuterole, muterole
Expected Arguments: <role ping/id>
Example Usage: >setmutedrole @Muted
Note: ping the role or give its ID.

Moderation Commands

This collection of commands has commands related with moderation. We consider someone a moderator if they have Manage messages permissions.

mute

This command allows you to mute someone for a certain time with a reason (or not).
Excpected Arguments: <member ping/id> <time/'forever'> <reason>
Exemple Usage: >mute @FaceFox 1h advertising
Note: You HAVE to set a mute role before running this command. Note 2: Mute the member by pinging or giving his id.

ban

This command allows you to ban someone for a certain time or forever from the server.
Excpected Arguments: <member ping/id> <time/'forever'> <reason>
Example Usage: >ban @FaceFox 7d DM advertising Note: Ban a member by pinging or giving his id.

kick

This command allows you to kick someone from the server.
Excpected Arguments: <member ping/id> <reason>
Example Usage: >kick @FaceFox DM advertising
Note: Kick a member by pinging or giving his id.

purge

You can delete up to 100 message from a channel (also from a user) with this command.
Excpected Arguments: <number of message> < user ping (optional)>
Example Usage: >purge 69
Note: You also can delete message from a member only.

simjoin / simban

You can simulate a join or a ban with these commands to test your configuration of the bot.
Example Usage: >simjoin
Note: These commands will not work if you don't setup the bot.

unmute / unban

This command allows you to unmute OR unban a member from your server.
Excpected Arguments: <user id>
Example Usage: >unmute @FaceFox
Note: You can't unban or unmute a user that has not been muted or banned from the server.

warn

Warn a member from your server for a specific reason, you can still check there warns afterwards.
Excpected Arguments: <member ping/id> <reason>
Example Usage: >warn @FaceFox spamming in general
Note: We consider someone a moderator if they have manage messages permissions. You can still require a role with another command.

list warnings

Display all warns for you or a specific member.
Aliases: warns, warnings, listwarns
Excpected Arguments: <member ping/id>
Example Usage: >warns @aero_noob
